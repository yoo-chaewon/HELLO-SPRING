### 1-1. 스프링 프레임워크

- 프레임워크란: 개발자들이 개발을 하기 위한 업무를 `추상적`으로 정의해놓은 `틀`

  - 수많은 기능들을 추상화 시켜놓고 대기중 -> 틀 -> 필요한기능만 구현하면 됨
  - Ex) 스프링 프레임워크, 안드로이드 개발 프레임워크

- 스프링 프레임워크는 주요기능으로 **DI, AOP, MVC, JDBC**(👉 어떠한 모듈을 만들어 나가는 방법론) 등을 제공한다.

  > DI: Dependency Injection - 의존성 주입, 기능을 만들어서 필요할 때마다 주입
  >
  > AOP: Aspect Oriented Programming - 관점지향 프로그래밍
  >
  > MVC: Model - View - Controller - 코드를 분리해서 구조화 하는 방법
  >
  > JDBC: Java DataBase Connectivity - 자바 데이터베이스 통신하는 방법

  

### 1-2. 스프링 프레임워크 모듈

- 스프링 프레임워크에서 제공하고 있는 모듈 👉 모듈화 되어 있다 !

  | 스프링 모듈  | 기능                                                         |
  | ------------ | ------------------------------------------------------------ |
  | Spring-core  | 스프링의 핵심인 DI와 IoC(Inversion of Control)를 제공        |
  | Spring-app   | AOP구현 기능 제공                                            |
  | Spring-jdbc  | 데이터베이스를 쉽게(적은 양의 코드) 다룰 수 있는 기능 제공   |
  | Spring-tx    | 스프링에서 제공하는 트랜잭션 관련 기능 제공                  |
  | Sring-webmvc | 스프링에서 제공하는 컨트롤러와 뷰를 이용한 스프링 MVC구현 기능 제공 |

  스프링 프레임워크에서 제공하고 있는 모듈을 사용하려면, 모듈에 대한 의존설정을 개발 프로젝트에 XML 파일등을 이용해서 개발자가 직접 하면 된다. 

  👉 spring-core, spring-jdbc ... 등 라이브러리를 내가 개발하고 있는 프로젝트에 삽입을 해서 기능 구현을 해야함.



### 1-3. 스프링 컨테이너(IoC)

- 스프링에서 객체를 생성하고 조립하는 컨테이너(Container)로, 컨테이너를 통해 생성된 `객체`를 `빈`(Bean)이라고 부른다.
  1. 객체생성 및 속성 데이터 작성(xml 문서)
  2. 스프링 `컨테이너`에서 `객체` 생성 및 조립(스프링 컨테이너 -> 빈생성 및 조립)
  3. 애플리케이션 구현(개발 문서에서 스프링 컨테이너에서 필요한 것 가져와 꺼내 씀)
